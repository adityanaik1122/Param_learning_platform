# Unity + C# Chapter Assignment Sheets

Source references:
- docs/Pro_Unity_Game_Development_with_C.pdf
- docs/Chamillard-C-Unity-Book-Code/Chapter1Code.zip to Chapter20Code.zip

Use one chapter per week for Weeks 1-20, then capstone in Weeks 21-24.

## Chapter 1 Assignment 
 
Reference code: `Chapter1Code.zip` 
 
### Problem Statement 
Implement, extend, and stabilize the baseline chapter project. Focus on correctness, code clarity, and production-ready behavior. 
 
### Required Tasks 
1. Baseline analysis: map scene objects to scripts and game flow. 
2. Guided feature: add one meaningful chapter-aligned mechanic. 
3. Reliability pass: add state guards, null checks, and reset handling. 
4. Refactor pass: improve naming and remove duplicated logic. 
5. Testing: run at least 8 manual test cases and log results. 
6. Demo: create 2-5 minute gameplay walkthrough. 
 
### Submission Checklist 
- `CHANGELOG_ch1.md` with technical summary 
- `TESTLOG_ch1.md` with test matrix and defect notes 
- Updated project files and demo video
